FEATURE ENVY

Sebuah method mengakses data dari objek lain lebih sering dibanding datanya sendiri.

CONTOH

class Order {
  private Customer customer;
  public Customer getCustomer() {
    return customer;
  }
  public double calculateTotal() {
    // Menggunakan data hanya dari Customer
    return customer.calculateTotalOrderPrice();
  }
}
class Customer {
  private List orders;
  public double calculateTotalOrderPrice() {
    double total = 0;
    for (Order order : orders) {
      total += order.getPrice();
    }
    return total;
  }
}

SOLUSI

class Order {
  private Customer customer;

  public Customer getCustomer() {
    return customer;
  }

  public double calculateTotal() {
    // Menggunakan data hanya dari Order sendiri
    return calculateTotalOrderPrice();
  }

  // Memindahkan method dari Customer sehingga dapat langsung diakses oleh Order
  private double calculateTotalOrderPrice() {
    double total = 0;
    for (Order order : orders) {
      total += order.getPrice();
    }
    return total;
  }
}

INAPROPRIATE INTIMACY

Sebuah class mengakses variabel internal dan berbagai method dari class lain.

CONTOH

class Customer {
  private String name;
  private Address address;

  public String getName() {
      return name;
  }

  public String getFullAddress() {
      // Mengakses secara langsung semua variabel internal dari Address
      return address.getStreet() + ", " + address.getCity() + ", " + address.getCountry();
  }
}

class Address {
  private String street;
  private String city;
  private String country;

  public String getStreet() {
      return street;
  }

  public String getCity() {
      return city;
  }

  public String getCountry() {
      return country;
  }
}

SOLUSI

class Customer {
  private String name;
  private Address address;

  public String getName() {
      return name;
  }

  public String getFullAddress() {
      // Sudah tidak secara langsung mengakses semua variabel internal dari Address
      return address.getFullAddress();
  }
}

class Address {
  private String street;
  private String city;
  private String country;

  public String getStreet() {
      return street;
  }

  public String getCity() {
      return city;
  }

  public String getCountry() {
      return country;
  }

  // Menambahkan method untuk mendapatkan address penuh, sehingga tidak perlu menggunakan
  // getters berkali-kali yang secara langsung mengakses variabel internal
  public String getFullAddress() {
      return street + ", " + city + ", " + country;
  }
}

MESSAGE CHAINS

Terjadi ketika meminta sebuah objek yang dimana objek yang diminta meminta objek lain dan objek lain ini juga meminta objek lain lagi dan seterusnya.

CONTOH

class A {
  B b;

  public String getInfo() {
      // Terjadi message chain
      return b.getC().getValue();
  }
}

class B {
  C c;

  public C getC() {
      return c;
  }
}

class C {
  private String value;

  public String getValue() {
      return value;
  }
}

SOLUSI

class A {
  B b;

  public String getInfo() {
      // Hanya memanggil satu method saja
      return b.getValueFromC();
  }
}

class B {
  C c;

  // Langsung memanggil c tanpa perlu memanggil getC(), dan kemudian langsung memanggil
  // valuenya, ini dilakukan sehingga tidak perlu menggunakan 2 method dalam memanggil value dari c pada
  // class A yang mengakibatkan message chain
  public String getValueFromC() {
      return c.getValue();
  }
}

class C {
  private String value;

  public String getValue() {
      return value;
  }
}

MIDDLE MAN

Sebuah class hanya melakukan satu aksi dan berperan sebagai perantara untuk mengakses method dari kelas lain.

CONTOH

public class MiddleMan {
  private OtherClass other;

  public MiddleMan() {
      this.other = new OtherClass();
  }

  public void doSomething() {
      // Memanggil method dari OtherClass, MiddleMan tidak punya fungsi lain selain ini
      other.methodToCall();
  }
}

public class OtherClass {
  public void methodToCall() {
      System.out.println("Method from OtherClass is called.");
  }
}

public class Main {
  public static void main(String[] args) {
      MiddleMan middleMan = new MiddleMan();
      middleMan.doSomething();
  }
}

SOLUSI

// Class MiddleMan dihapus karena memiliki fungsi yang redundan, methodToCall() dapat langsung
// dipanggil dari Main tanpa perlu melewati MiddleMan

 public class OtherClass {
    public void methodToCall() {
        System.out.println("Method from OtherClass is called.");
    }
}
  
public class Main {
    public static void main(String[] args) {
        OtherClass other = new OtherClass();
        other.methodToCall();
    }
}

INCOMPLETE LIBRARY CLASS

Penggunaan library tanpa fitur-fitur yang memenuhi kebutuhan dan tidak dapat dimodifikasi untuk memenuhi kebutuhan karena properti library adalah read-only.

CONTOH

// Misalkan ReadOnlyLibrary adalah library read-only, dan tidak memiliki fitur yang diperlukan,
// sehingga perlu menambahkan fiturnya ke library, namun karena properti library read-only,
// tidak dapat mengubah isi dari library, sehingga tidak dapat menambahkan fitur yang diperlukan
public class ReadOnlyLibrary {
    private String data;
  
    public ReadOnlyLibrary(String data) {
        this.data = data;
    }
  
    public String getData() {
        return data;
    }
}

SOLUSI

// Solusinya membuat child class dari ReadOnlyLibrary, sehingga bisa menambahkan
// fitur yang diperlukan ke library, tanpa perlu akses ke ReadOnlyLibrary
public class ExtendedLibrary extends ReadOnlyLibrary {
    public ExtendedLibrary(String data) {
        super(data);
    }
  
    public void writeData(String newData) {
        System.out.println("Menulis data baru: " + newData);
    }
}
